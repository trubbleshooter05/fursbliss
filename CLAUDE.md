
# Avatar-Driven Development

Before product, UX, copy, onboarding, pricing, SEO, content, analytics, conversion, or customer-facing feature work:

1. Read `.cursor/rules/avatar-checklist.mdc`.
2. If the work is customer-facing, also read `.cursor/rules/avatar-driven-development.mdc`.
3. Identify the matching avatar before proposing or coding.
4. Do not build vague features for vague users.
5. Prefer revenue, conversion, retention, traffic, or engagement impact.
